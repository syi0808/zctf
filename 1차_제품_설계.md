# zctf 1차 제품화 상세 설계서

## Rust Macro → Schema Artifact → Rust Codegen → JS/TS View → Transport Adapter

## 1. 목표

`zctf`의 1차 제품화 목표는 Rust-native JavaScript tooling에서 재사용 가능한 **schema-driven binary interop SDK**를 만드는 것이다.

핵심 사용 경험은 다음과 같다.

```rust
#[zctf::document]
pub struct TransformResult {
    #[zctf(string(direct, encoding = "utf8"))]
    pub code: String,

    pub duration_ms: f64,
}
```

위 Rust type으로부터 다음을 생성한다.

```txt
1. zctf schema artifact
2. Rust encode/decode helper
3. JavaScript View class
4. TypeScript declaration
5. N-API / WASM / FFI transport helper와 연결 가능한 ZctfDocument trait impl
```

중요한 원칙은 다음이다.

```txt
zctf macro:
  Rust type을 zctf schema/layout으로 변환한다.

zctf-codegen:
  schema artifact에서 Rust/JS/TS output을 생성한다.

transport adapter:
  encode된 zctf binary document를 각 runtime에 맞게 감싼다.
```

`zctf` core는 `svgr2`, SVG, JSX, config, plugin 같은 도메인 스펙을 알지 않는다.

## 2. 1차 범위

## 2.1 포함

```txt
- Rust #[zctf::document] macro
- Rust #[zctf::record] macro
- Rust #[zctf::enum_repr] macro
- schema artifact 생성
- zctf-codegen Rust crate
- JS View codegen
- TS declaration codegen
- Rust helper codegen
- ZctfDocument trait
- ZctfWriter
- manual writer API
- N-API Buffer helper
- WASM ptr/len/cap helper
- FFI owned ptr/len/cap helper
- 최소 validator
- macro encode vs manual writer benchmark
```

## 2.2 제외

```txt
- TypeScript interface importer
- JS 기반 codegen engine
- FunctionRef / callback bridge
- patch protocol
- schema evolution
- recursive graph
- complex generic
- data-carrying enum
- external string optimization
- dynamic Proxy view
- full plugin bridge
```

1차 목표는 “완전한 프레임워크”가 아니라 다음이다.

```txt
Rust type 하나에서
schema artifact, generated JS view, generated TS type, binary encode path까지
일관되게 생성되는 최소 제품화 기반.
```

## 5. zctf-core 상세 설계

## 5.1 핵심 trait

```rust
pub trait ZctfDocument {
    const SCHEMA_ID: u64;
    const LAYOUT_VERSION: u32;

    fn encode_zctf(&self, writer: &mut ZctfWriter) -> ZctfResult<()>;
}
```

1차에는 decode trait을 선택사항으로 둔다.

```rust
pub trait ZctfDecode<'a>: Sized {
    fn decode_zctf(doc: ZctfBorrowedDocument<'a>) -> ZctfResult<Self>;
}
```

## 5.2 public API

```rust
pub fn encode_owned<T: ZctfDocument>(value: &T) -> ZctfResult<Vec<u8>> {
    let mut writer = ZctfWriter::new();
    value.encode_zctf(&mut writer)?;
    writer.finish()
}
```

사용 예시:

```rust
let bytes = zctf::encode_owned(&result)?;
```

## 5.3 Document header

1차 binary document header는 고정 크기로 둔다.

```txt
DocumentHeader:
  magic: u32
  format_version: u16
  endian: u8
  flags: u8
  schema_id: u64
  layout_version: u32
  root_offset: u32
  string_table_offset: u32
  string_heap_offset: u32
  document_len: u32
```

권장 값:

```txt
magic:
  "ZCTF" as u32

endian:
  1 = little endian

format_version:
  1
```

## 5.4 String table layout

1차 string table entry:

```txt
StringEntry:
  offset: u32
  byte_len: u32
```

offset은 string heap 시작 기준 relative offset이다.

확장을 고려한 v2 후보:

```txt
StringEntryV2:
  offset: u32
  byte_len: u32
  char_len: u32
  flags: u32
```

1차에서는 `StringEntryV1`만 구현한다.

## 5.5 List layout

1차 list는 fixed-stride list만 지원한다.

```txt
ListHeader:
  len: u32
  stride: u32
  items_offset: u32
  reserved: u32
```

`Vec<Record>`는 이 layout으로 encode된다.

## 6.3 schema id

1차 schema id는 canonical schema artifact의 stable hash로 계산한다.

```txt
schema_id = xxhash64(canonical_schema_json)
```

또는 `blake3` 기반 64-bit truncate를 사용한다.

중요한 조건:

```txt
field order, type, encoding, layout policy가 바뀌면 schema_id가 바뀐다.
```

## 7. Rust Macro 상세 설계

## 7.1 document macro

입력:

```rust
#[zctf::document]
pub struct TransformResult {
    #[zctf(string(direct, encoding = "utf8"))]
    pub code: String,

    pub duration_ms: f64,

    pub warnings: Vec<Warning>,
}
```

macro 역할:

```txt
1. Rust struct AST 파싱
2. field type을 TypeIr로 변환
3. #[zctf(...)] attribute 파싱
4. SchemaIr 생성
5. schema artifact emit
6. ZctfDocument impl 생성
```

생성 코드 개념:

```rust
impl zctf::ZctfDocument for TransformResult {
    const SCHEMA_ID: u64 = 0x91f2_a4c1_0000_0001;
    const LAYOUT_VERSION: u32 = 1;

    fn encode_zctf(&self, writer: &mut zctf::ZctfWriter) -> zctf::Result<()> {
        let root = writer.begin_document(Self::SCHEMA_ID, Self::LAYOUT_VERSION);

        writer.set_string(root, FIELD_CODE, &self.code)?;
        writer.set_f64(root, FIELD_DURATION_MS, self.duration_ms)?;
        writer.set_list(root, FIELD_WARNINGS, &self.warnings)?;

        writer.finish_document(root)
    }
}
```

## 9.3 JS output

파일:

```txt

```

생성 내용:

```js
import {
  BinaryDocument,
  FixedListView,
} from '@zctf/runtime'

const SCHEMA_ID = 0x91f2a4c100000001n
const LAYOUT_VERSION = 1

const FIELD_CODE = 0
const FIELD_DURATION_MS = 8
const FIELD_WARNINGS = 16

export class TransformResultView {
  constructor(document, offset) {
    this.document = document
    this.offset = offset
  }

  static from(bytes) {
    const document = BinaryDocument.from(bytes, {
      schemaId: SCHEMA_ID,
      layoutVersion: LAYOUT_VERSION,
    })

    return new TransformResultView(document, document.rootOffset)
  }

  get code() {
    return this.document.stringAt(this.offset + FIELD_CODE)
  }

  get durationMs() {
    return this.document.f64(this.offset + FIELD_DURATION_MS)
  }

  get warnings() {
    const offset = this.document.u32(this.offset + FIELD_WARNINGS)
    return new WarningListView(this.document, offset)
  }

  toObject() {
    return {
      code: this.code,
      durationMs: this.durationMs,
      warnings: this.warnings.toArray(),
    }
  }
}
```

## 10.1 BinaryDocument

```js
  constructor(bytes, options) {
    this.bytes = normalizeBytes(bytes)

    return new BinaryDocument(bytes, options)
  }

  u8(offset) { ... }
  u16(offset) { ... }

```

## 10.2 String read path

1차 기본:

```js
string(id) {
  const entry = this.stringTableOffset + id * 8
  const start = this.stringHeapOffset + this.u32(entry)
  const len = this.u32(entry + 4)

  return this.buffer.toString('utf8', start, start + len)
}
```

Node Buffer가 아닌 `Uint8Array`에서는 fallback:

```js
string(id) {
  const bytes = this.bytes.subarray(start, start + len)
  return decoder.decode(bytes)
}
```

1차 runtime은 Node 우선으로 `Buffer.toString` path를 최적화한다.

## 10.3 FixedListView

```js
export class FixedListView {
  constructor(document, offset, stride, createItem) {
    this.document = document
    this.offset = offset
    this.stride = stride
    this.createItem = createItem

    this._length = document.u32(offset)
    this.itemsOffset = document.u32(offset + 8)
  }

  get length() {
    return this._length
  }

  get(index) {
    if (!Number.isInteger(index) || index < 0 || index >= this._length) {
      throw new RangeError('list index out of bounds')
    }

    return this.createItem(
      this.document,
      this.itemsOffset + index * this.stride,
      index,
    )
  }

  at(index) {
    if (index < 0) index = this._length + index
    if (index < 0 || index >= this._length) return undefined
    return this.get(index)
  }

  forEach(callback) {
    for (let i = 0; i < this._length; i++) {
      callback(this.get(i), i)
    }
  }

  map(callback) {
    const out = new Array(this._length)
    for (let i = 0; i < this._length; i++) {
      out[i] = callback(this.get(i), i)
    }
    return out
  }

  toArray() {
    return this.map((item) => item.toObject())
  }

  *[Symbol.iterator]() {
    for (let i = 0; i < this._length; i++) {
      yield this.get(i)
    }
  }

  forEachRaw(callback) {
    let offset = this.itemsOffset
    for (let i = 0; i < this._length; i++) {
      callback(offset, i, this.document)
      offset += this.stride
    }
  }
}
```

## 11. Manual Writer 상세 설계

## 11.1 High-level writer

```rust
let mut writer = ZctfWriter::new();

let root = writer.begin_document(TRANSFORM_RESULT_LAYOUT);

writer.set_string(root, FIELD_CODE, "export default Icon")?;
writer.set_f64(root, FIELD_DURATION_MS, 1.2)?;

let warnings = writer.begin_list(root, FIELD_WARNINGS, 0)?;
writer.finish_list(warnings)?;

writer.finish_document(root)?;

let bytes = writer.finish()?;
```

## 11.2 Lower-level writer

```rust
let mut writer = ZctfWriter::with_capacity(1024);

let code_ref = writer.write_string("export default Icon")?;
let warnings_ref = writer.write_fixed_list::<WarningRecord>(&[])?;

writer.write_root_ref(FIELD_CODE, code_ref)?;
writer.write_root_f64(FIELD_DURATION_MS, 1.2)?;
writer.write_root_ref(FIELD_WARNINGS, warnings_ref)?;

let bytes = writer.finish()?;
```

## 11.3 Writer internal phases

```txt
Phase 1:
  reserve header

Phase 2:
  write root record

Phase 3:
  write nested records/lists

Phase 4:
  write string table

Phase 5:
  write string heap

Phase 6:
  patch header offsets

Phase 7:
  finish Vec<u8>
```

1차는 단순성을 위해 writer가 내부적으로 append-only sections를 관리한다.

```rust
pub struct ZctfWriter {
    header: HeaderBuilder,
    records: Vec<u8>,
    lists: Vec<u8>,
    string_table: Vec<StringEntry>,
    string_heap: Vec<u8>,
}
```

### helper 구현

```rust
pub fn to_buffer<T: zctf::ZctfDocument>(
    value: &T,
) -> napi::Result<napi::bindgen_prelude::Buffer> {
    let bytes = zctf::encode_owned(value)
        .map_err(|err| napi::Error::from_reason(err.to_string()))?;

    Ok(bytes.into())
}
```

## 18. 최종 요약

1차 zctf는 다음 경험을 제공해야 한다.

```rust
#[zctf::document]
struct Foo {
    name: String,
    size: u32,
}
```

여기서 자동으로:

```txt
- schema artifact
- Rust encode implementation
- JS View class
- TS declaration
```

이 생성된다.

그리고 transport는 다음처럼 얇게 붙는다.

```rust
#[napi]
pub fn make_foo() -> napi::Result<Buffer> {
    zctf_napi::to_buffer(&foo)
}
```

또는:

```rust
#[zctf_napi::export(return = "buffer")]
pub fn make_foo() -> Foo {
    foo
}
```

핵심 설계 원칙은 다음이다.

```txt
zctf macro는 schema/layout을 만든다.
zctf-codegen Rust crate는 Rust/JS/TS output을 만든다.
transport adapter는 binary document를 각 runtime에 맞는 representation으로 감싼다.
manual writer는 macro path를 검증하고 low-level 최적화 실험을 가능하게 한다.
```

이 구조를 따르면 `zctf`는 특정 도메인을 몰라도, `svgr2` 같은 Rust-native JavaScript tooling이 자기 Rust type만 정의해서 사용할 수 있는 범용 binary interop SDK가 된다.
